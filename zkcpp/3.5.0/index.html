Quickstart

Dependencies

Install all the dependencies:

  • git
  • ant
  • cmake
  • log4cxx
  • boost with thread, system, and filesystem libraries
  • googletest

Building and Installing

Clone the git repository:

git://github.com/m1ch1/zookeeper.git

This will create a local repository under zookeeper/ directory.

cd zookeeper/
ZKHOME=`pwd`
ant

Now, compile the C++ client and install:

cd $ZKHOME/src/contrib/zkcpp
mkdir build
cd build
cmake ..
make
sudo make install

This will install a shared library libzkcpp.so under /usr/local/lib and include headers under /usr/local/include/zookeeper.

Starting a ZooKeeper Server in Standalone mode

cp $ZKHOME/conf/zoo_sample.cfg $ZKHOME/conf/zoo.cfg
$ZKHOME/bin/zkServer.sh start
echo ruok | nc localhost 2181

You can find more information about the standalone mode in ZooKeeper Getting Started Guide.

Hello, world!

Now let's compile and run our first program, examples/zk_hello_world.cc. This program creates a znode, reads from the znode and prints the value, and removes the znode. 

cd $ZKHOME/src/contrib/zkcpp/examples
g++ zk_hello_world.cc -o zk_hello_world -I /usr/local/include -L /usr/local/lib -lzkcpp
./zk_hello_world

It should output something like this: 

log4cxx: No appender could be found for logger (zookeeper.c).
log4cxx: Please initialize the log4cxx system properly.
/hello world

The Logging section describes how to configure log4cxx logging to get rid of this warning.

Logging

Zkcpp uses Apache log4cxx for logging. There are many ways to configure log4cxx, but the simplest way is to put a log4cxx configuration file named log4cxx.properties under the same directory as your executable that links against zkcpp. Here is a sample configuration that creates a log file test.log with the log level set to DEBUG: 

log4j.rootLogger=DEBUG, file
log4j.appender.file=org.apache.log4j.RollingFileAppender
log4j.appender.file.File=test.log
log4j.appender.file.MaxFileSize=100MB
log4j.appender.file.MaxBackupIndex=1
log4j.appender.file.layout=org.apache.log4j.PatternLayout
log4j.appender.file.layout.ConversionPattern=%d [%t] %-5p %c:%L %m%n

Run zk_hello_world after creating a log4cxx.properties with the above configuration in the same directory as zk_hello_world. This time, you should not see the log4cxx warnings, and there is a log file called test.log that contains debug logs like these:

Znode Stat

Some ZooKeeper operations return a data::Stat object, which contains metadata information about a znode. The Stat contains the following fields (from ZooKeeper Programmer's Guide):

  • czxid The zxid of the change that caused this znode to be created.
  • mzxid The zxid of the change that last modified this znode.
  • ctime The time in milliseconds from epoch when this znode was created.
  • mtime The time in milliseconds from epoch when this znode was last modified.
  • version The number of changes to the data of this znode.
  • cversion The number of changes to the children of this znode.
  • aversion The number of changes to the ACL of this znode.
  • ephemeralOwner The session id of the owner of this znode if the znode is an ephemeral node. If it is not an ephemeral node, it will be zero.
  • dataLength The length of the data field of this znode.
  • numChildren The number of children of this znode.

You can pass the znode version to set() and remove() operations, and aversion to setAcl() operation to avoid conflicting updates. These operations will fail with ReturnCode::BadVersion return code if the (a)version passed to these operation does not match the current znode (a)version. You can pass -1 to these operations to bypass the version check.
